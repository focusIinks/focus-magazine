1. PAGE SPEED — Score: 3/10
-------------------------------------------------------------------------------
The SimDif-built page is a performance disaster. The HTML document alone is
135 KB across 587 lines — and approximately 95% of that weight is inline CSS
bloated with the entire SimDif framework stylesheet, including classes for
e-commerce, galleries, blogs, and features not even used on this single-page
site. The <style> block starting at line 46 is a monolithic wall of
unminified, unpurged CSS that ships to every single page load.

External resources load from cdnjs.cloudflare.com (jQuery 1.12.4 deferred,
jQuery UI 1.12.1 deferred, Swiper 11.0.5 SYNCHRONOUS), a custom sd_js.js
file (deferred), and an external sd_style.css. The Swiper CSS and JS bundle
(~80 KB combined) is loaded synchronously without defer or async, creating a
render-blocking bottleneck. Combined with the massive inline CSS and the
font-face declarations for Nunito Sans (5 normal + 4 italic subsets) and
Quattrocento (4 subsets), the critical rendering path is severely congested.

The fixed background image (background-size: 1920px 642px) is forced to
1920px width regardless of viewport, wasting bandwidth on mobile devices. The
CSS blur filter applied via ::after pseudo-element on the background wrapper
adds GPU compositing overhead on low-end devices. The CSS animation-based
mobile navigation (mobilenavopen, mobilenavclose, etc.) and the backdrop-filter
blur effects create additional paint compositing work.

Estimated Lighthouse Performance score: 25-40 out of 100.

CRITICAL: Swiper is loaded synchronously but there are zero Swiper carousels
visible in the HTML body. This is dead weight — a full slider framework loaded
for nothing.


2. MOBILE OPTIMIZATION — Score: 5/10
-------------------------------------------------------------------------------
The site uses responsive design with multiple media query breakpoints
(min-width: 348px, 376px, 425px, 450px, 500px, 553px, 558px, 601px, 667px,
720px, 768px, 796px, 896px, 965px, 960px). However, this is clearly a
desktop-first approach — the base styles apply to mobile, and the @media
(min-width) queries progressively enhance for larger screens. True mobile-first
would use max-width queries as the foundation.

The mobile navigation works via slide-in animation from the left (80% width
sidebar), which is functional but not modern. Touch-action: manipulation is
correctly set, and -webkit-overflow-scrolling: touch is present for iOS
momentum scrolling.

The viewport meta tag is correctly configured: width=device-width,
initial-scale=1. The line-height is set to 1.2, which is tight for body text
accessibility (WCAG recommends 1.5 for body text).

The fixed header background at 1920px is not responsive to screen size — it
will be downscaled on mobile but still downloads at full resolution. The
"superphone" layout variant at 667px+ shows the platform tries to accommodate
larger phones, which is a plus.

Partially responsive, but far from optimized for mobile performance.

INP (Interaction to Next Paint): LIKELY POOR

  - No evidence of efficient event delegation.


4. JAVASCRIPT — Score: 2/10

This is the most alarming technical finding:

  a) jQuery 1.12.4 (May 2016) — loaded via defer from cdnjs.
     - 10 YEARS OLD. Contains known security vulnerabilities:
       * CVE-2020-11022: Cross-site scripting via .html()
       * CVE-2020-11023: Cross-site scripting via .htmlPrefilter()
       * CVE-2019-11358: Prototype pollution via $.extend(true)
       * CVE-2020-23064: Regex DoS in jQuery.htmlPrefilter
     - End of life since jQuery 1.x was deprecated in 2020.
     - 97 KB minified — massive overhead for what the site actually does.

  b) jQuery UI 1.12.1 (September 2016) — loaded via defer from cdnjs.
     - Also 10 years old with known vulnerabilities.
     - 232 KB minified. Used for... select menu styling? The language
       selector? This is criminal overhead for a basic dropdown.

  c) Swiper 11.0.5 — loaded SYNCHRONOUSLY from cdnjs.
     - This is the only current library on the page.
     - BUT THERE IS NO SWIPER CAROUSEL IN THE HTML. Dead weight.
     - ~80 KB for CSS + JS combined, render-blocking, for nothing.

  d) sd_js.js — SimDif's custom script, deferred with cache-busting hash.
     - Black box. No way to audit its contents from outside.
     - Likely handles navigation, readmore blocks, contact form, etc.

  e) Cloudflare email-decode script — for obfuscating email addresses.

Total JS payload estimate: ~410 KB (minified) — absolutely devastating for
