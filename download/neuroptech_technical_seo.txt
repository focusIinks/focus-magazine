2.1 PLATFORM: LOVABLE (LOW-CODE) — CRITICAL LIMITATIONS
─────────────────────────────────────────────────────────────────────────────────
  [CRITICAL] The site is built on Lovable, an AI-powered low-code platform.
  This creates fundamental, often insurmountable limitations:

  ARCHITECTURE:
    - Client-Side Rendered (CSR) React SPA
    - NO Server-Side Rendering (SSR)
    - NO Static Site Generation (SSG)
    - NO pre-rendering configured (evidenced by empty HTML shell in source)
    - Single HTML document served for ALL routes

  WHAT THIS MEANS FOR CRAWLERS:
    - Google sees an EMPTY HTML page with only <script> and <style> tags
    - The actual content ("Welcome To NEUROPTECH", all sections, etc.) is
      generated by JavaScript AFTER the page loads
    - While Google can execute JavaScript (Googlebot can render JS), it:
      a) Takes longer to index (second wave of indexing)
      b) May miss content if JS execution times out
      c) Cannot follow JS-generated links reliably
      d) Cannot index client-side-only pages properly

  WHAT THIS MEANS FOR USERS:
    - Slower initial page load (must wait for JS download + execution)
    - No content visible if JavaScript fails
    - Poor experience on slow connections
    - Higher bounce rate probability

  Lovable's OWN DOCUMENTATION acknowledges this limitation:
    "CSR limitations: crawlers may see empty HTML (hurts SEO), slower initial
     load (harms Core Web Vitals and rankings)"
    Source: lovable.dev/faq/deployment/rendering/csr-seo-limitations
