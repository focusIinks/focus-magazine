## 2.6 ACCESSIBILITY — Score: 2/10

This is genuinely awful.

**Critical Violations:**

1. **Hidden scrollbars** — Both sites use `no-scrollbar` / `scrollbar-width: none` / `overflow-x-hidden`. This is an accessibility violation. Scrollbars communicate scrollability to users. Removing them is an aesthetic choice that harms usability.

2. **No ARIA landmarks** — Neither site uses `<nav>`, `<main>`, `<aside>`, `<section role="...">`, or any ARIA attributes. Screen readers cannot navigate these sites.

3. **No skip navigation links** — Standard accessibility requirement completely absent.

4. **No alt text structure** — While some images have alt attributes (visible in JS bundle), many use generic descriptions like "Customer UI" and "NexPOS Feature" — not descriptive enough for screen readers.

5. **Low contrast body text** — OptoBharat uses `color: var(--color-foreground)` with `#e9ebed` on `#0b0c0f` — while this passes WCAG AA for large text, the smaller muted text (`color: var(--color-muted-foreground): #9da4af`) on dark backgrounds may fail AA for normal text.

6. **No visible focus indicators** — No custom focus styles detected in either CSS file.

7. **Form inputs lack visible labels** — Contact form fields (Phone Number, Email) have placeholder text but no associated `<label>` elements detected.

8. **No language declaration consistency** — OptoBharat supports 7+ languages but the HTML `lang="en"` is hardcoded. Dynamic language switching doesn't update the document language.

## FATAL FLAW #2: 4.5MB JavaScript Bundle
**Severity: CRITICAL (Performance)**

OptoBharat's main JS bundle is **4,488,742 bytes (4.3MB)**. This is approximately:
- 10x the recommended bundle size
- Will take 30-60 seconds to load on a 4G connection in India
- Will take 5-10 seconds to parse and execute on a mid-range phone

## FATAL FLAW #3: No SEO — Zero Indexable Content
**Severity: CRITICAL (Discovery)**

Both sites are Single Page Applications that render all content client-side. The raw HTML of both sites contains:

- Envision Bharat: `<div id="root"></div>` — Same. Zero indexable content.

## FATAL FLAW #6: Tailwind CDN in Production
**Severity: HIGH (Performance/Reliability)**

Envision Bharat loads Tailwind CSS via `<script src="https://cdn.tailwindcss.com">`. This is explicitly documented by Tailwind as **development only**:
- It's ~300KB+ of JavaScript (not CSS)
- It generates CSS at runtime in the browser
- It can be blocked by ad blockers and corporate firewalls
- It adds significant page load time
- It's not cached effectively
- If cdn.tailwindcss.com goes down, the site breaks

---

## FATAL FLAW #7: ESM Imports from esm.sh in Production
**Severity: HIGH (Reliability)**

Envision Bharat loads React, React DOM, and Lucide icons from `esm.sh`:
```json
{
  "imports": {
    "react": "https://esm.sh/react@^19.2.3",
    "react-dom": "https://esm.sh/react-dom@^19.2.3",
    "lucide-react": "https://esm.sh/lucide-react@^0.561.0"
  }
}
```

This means:
- Three additional HTTP requests to a third-party CDN on every page load
- No guarantee of availability
- No control over version pinning (`^19.2.3` allows minor version bumps)
- esm.sh could be blocked by network policies in India
- Adds latency to initial render
